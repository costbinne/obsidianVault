# 前端
## 页面原型
最开始的时候制作本次所需的template
### 常用工具介绍
#### Figma
igma 是一个“界面设计工具”，不是写代码的
- 画页面结构（布局）
    
- 设计按钮、输入框、颜色、间距
    
- 预览手机 / PC 的样子
    
- 和别人讨论“页面长什么样”
    

👉 **Figma 产出的是：设计稿，不是网页**
#### 你在 Figma 里做的事是：

- 拖一个按钮
    
- 拖一个卡片
    
- 调间距、颜色
    
- 看手机效果
适合谁？
- UI / 设计导向项目
    
- 团队里有“设计 → 前端”的分工
    
- 要反复讨论样式

#### Bootstrap
Bootstrap 是一套“现成的 UI 零件库”

它已经帮你设计好了：

- 按钮
    
- 表单
    
- 卡片
    
- 布局（手机自适应）
Bootstrap 怎么用？
```
<button class="btn btn-primary">登录</button>
```

Bootstrap 的特点

✅ 上手非常快  
✅ 手机端自适应天然支持  
✅ 几乎不用写 CSS  
✅ 和 Thymeleaf / JSP / Spring **完美兼容**

 适合谁？

- 后端工程师
    
- 企业系统
    
- 快速做可用页面
    
- 不想研究 CSS

#### Tailwind
“不提供现成样式，只给你工具”
Tailwind 是一个“把 CSS 原子化，直接写在 HTML class 里的工具”。

Example
```
<div class="w-full p-4 bg-gray-100 text-center">
```
每一个 class：

- 都是一个 CSS 属性
    
- 非常细粒度

Tailwind 的特点

✅ 样式自由度极高  
✅ 很“现代”、像写配置  
❌ class 非常长  
❌ 初期写得慢  
❌ 对 CSS 理解要求高

 适合谁？

- 前端为主
    
- 对样式有追求
    
- 有时间打磨 UI
    
- 熟悉 CSS

##### Tailwind的优势
- **样式原子化（Utility-first）**  
    每个 class 对应一个明确的 CSS 属性，样式直观、可预测，不依赖隐藏的全局样式。
    
- **无需命名 CSS class**  
    不需要想 `.login-box`、`.card-wrapper` 这种名字，减少认知负担和命名冲突。
    
- **样式与结构强关联**  
    HTML 就能完整描述页面外观，不需要在 HTML 与 CSS 文件之间来回跳转。
    
- **高度可组合**  
    任意样式通过 class 组合完成，灵活度极高，适合复杂或差异化界面。
    
- **一致性强**  
    间距、颜色、字号、阴影等都来自统一设计系统，页面风格天然统一。
    
- **响应式设计简单直观**  
    通过 `sm:` / `md:` / `lg:` 等前缀即可实现多端适配，逻辑清晰。
    
- **无需写自定义 CSS（大多数场景）**  
    减少 CSS 文件数量，降低样式层维护成本。
    
- **避免 CSS 全局污染问题**  
    不依赖全局 class 或层叠规则，几乎不存在“改 A 影响 B”的情况。
    
- **非常适合组件化开发**  
    与 React / Vue / 组件模板体系天然契合，样式可随组件复用。
    
- **生产构建体积小**  
    配合 purge / JIT，仅保留实际使用的样式，最终 CSS 非常精简。
    
- **样式可读性对熟练者极高**  
    熟悉 Tailwind 后，一眼就能看懂一个组件的外观结构。
    
- **修改样式速度快**  
    改动只在一行 HTML 中完成，调试与迭代速度极高。
    
- **无需深度理解 CSS 选择器规则**  
    减少 `:hover`、`>`, `+`, 层叠、优先级等复杂问题。
    
- **设计系统可配置**  
    颜色、间距、字号可在配置中统一管理，利于长期维护。
    
- **现代前端生态主流方案**  
    在现代前端项目（SaaS、产品型项目）中广泛使用，社区活跃。
