- **Eclipse**
    
    - Maven
        
    - Debug
        
    - Tomcat
        
    - 跑单测
        
- **VS Code + Codeium**
    
    - 写代码
        
    - 改配置
        
    - 写 SQL / HTML
        
    - AI 辅助生成

其余需要的插件
##### 1) Extension Pack for Java（微软）

**用途**：Java 语法/跳转/重构/调试基础能力  
**理由**：你会在 VS Code 里写 Java，没它体验会差很多。
##### 2) Spring Boot Extension Pack（Pivotal / VMware）

**用途**：Spring Boot 配置提示、运行配置、依赖导航、properties/yaml 支持  
**理由**：你大量会写 `application-*.yml`、Security 配置，提示和跳转很值。

##### 3) YAML（Red Hat）

**用途**：YAML 校验 + 自动补全 + 格式化  
**理由**：你分环境配置（dev/prod）会频繁改 yml，这个能减少低级错误。

##### 4) SQLTools + MySQL Driver（或你喜欢的 DB 插件）

**用途**：在 VS Code 里连 MySQL、跑 SQL、看表数据  
**理由**：配合 Flyway/调试数据超方便，不用频繁切工具。

##### 9) Markdown All in One

**用途**：写 README / 设计说明更顺手  
**理由**：你会写技术栈、要件、操作手顺，这个省事。
#### 关于 Codeium
如果出现 无法找到codeium
##### 代替方案 装「Windsurf Plugin」（就是原 Codeium）
Codeium 现在很多地方是以 **Windsurf（旧 Codeium）** 的名字发布在 Eclipse Marketplace 里的。
- Eclipse 菜单：**Help → Eclipse Marketplace…**
    
- 搜索框输入：`Windsurf`
    
- 找到 **Windsurf Plugin**（描述里一般会写 formerly Codeium / 旧 Codeium）
    
- 点 **Install**
    
- 重启 Eclipse
    
- 按提示 **Sign in / Login**（浏览器授权）
##### 方案 B：继续用 “Install New Software…”（手动 Update Site）

如果你就是想走 update site，那常见可用的是这个（注意末尾的 `/`）：

**`https://exafunction.github.io/codeium-eclipse/`**

**操作步骤**
1. **Help → Install New Software…**
    
2. **Add…**
    
    - Name: `Codeium`
        
    - Location: `https://exafunction.github.io/codeium-eclipse/`
        
3. 勾选插件 → Next → Finish → 重启
    
4. 登录授权
    

> 这个站点在浏览器打开可能显示 NOT FOUND，但它的 update site 文件（比如 `content.jar`）是存在的，Eclipse 能识别。
