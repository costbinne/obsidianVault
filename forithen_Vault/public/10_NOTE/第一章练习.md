## IGW
默认的IGW和VPC的关系
![[Pasted image 20260503130244.png]]可以看到IGW只和默认VPC做了关联 
所以 IGW 是 VPC级别的资源
IGW 不是属于 subnet，而是挂在整个 VPC 上

默认 IGW  
↓  
Attached  
↓  
某个 VPC ID
这个IGW已经连接到那个VPC

VPC  
├─ Subnet  
├─ Route Table  
└─ IGW

### 为什么 IGW 要是 VPC级别？
VPC 是整个网络边界
IGW 本质是：这个VPC通向互联网的出口 它应该挂在整个VPC上
但是即便 VPC挂了IGW 也不代表所有subnet能自动上网

真正决定某个subnet能不能上网的是 Route Table

### keypair
![[Pasted image 20260504204514.png]]
![[Pasted image 20260504204503.png]]
Key pair 是你登录 EC2 的“钥匙”。
Public key：公钥  
Private key：私钥
AWS 会把 public key 放到 EC2 里面。  
你下载到自己电脑上的 `.pem` 文件，就是 private key。

之后你从自己电脑 SSH 连接 EC2 时，流程是：
```
你的电脑拿 private key  
↓  
EC2 里有对应 public key  
↓  
匹配成功  
↓  
允许登录
```

所以 key pair 不是密码，而是一种“密钥登录”机制。  
`.pem` 文件一定要保存好，丢了之后不能重新下载同一个私钥。
```
Key pair name: study-key  
Key pair type: RSA  
Private key file format: .pem
```
RSA 现在完全可以用，`.pem` 适合 OpenSSH，也适合你之后用 Windows PowerShell / Git Bash / VS Code SSH 登录。

#### RSA
RSA = 一种“加密算法”，用来生成密钥对（公钥 + 私钥）
生成一对密钥：  
  
公钥（public key） → 放到EC2服务器里  
私钥（private key） → 下载给你（.pem文件）

登录时发生什么？
你执行：
ssh -i study-key.pem ec2-user@IP
